Follow-up: byte stack argument page-boundary fix
Date: 2026-07-08
Problem after fixture restore:
- Restored fixture: `tests/tests/chapter_16/valid/chars/data_on_page_boundary_linux.s` places `zed` as the last byte of a page.
- Official gate reached real product failure: `valid/chars/push_arg_on_page_boundary` expected 1, got -11 (segfault).
- Root cause: stack-passed byte arguments used generic `pushq <memory operand>`, causing an 8-byte read from a one-byte `char` object at page end.

Fix:
- Updated `src/codegen/codegen.rs` `lower_call` stack-argument lowering.
- For `OperandType::Byte`, emit `movsbl src, %r10d` followed by `pushq %r10`.
- For `OperandType::UByte`, emit `movzbl src, %r10d` followed by `pushq %r10`.
- Non-byte stack arguments keep the existing `pushq` path.
- This keeps ABI stack slots 8 bytes wide while ensuring byte source operands are read with byte-width instructions only.
