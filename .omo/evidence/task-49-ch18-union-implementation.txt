Implementation summary
- Added native Type::Union and AggregateKind::{Struct, Union}; struct declarations now carry aggregate kind while preserving existing StructDecl/TopLevelItem plumbing.
- Parser accepts union type specifiers, forward declarations, and definitions; union specifiers are rejected when combined with other type specifiers.
- Resolver tracks one tag namespace with aggregate kind, so struct/union same-tag conflicts in the same scope are rejected and inner same-name different-kind declarations shadow outer tags correctly.
- Type table stores aggregate kind plus members/order/size/alignment. Struct layout remains padded-offset layout. Union layout assigns every member offset 0, size=max member size rounded to max alignment, alignment=max member alignment.
- Typechecker accepts/rejects union declarations, incomplete unions, member access, initialization, assignment/copy, conditional expressions, pointer comparisons, casts to void, and invalid scalar use consistently with the Chapter 18 union tests.
- Lowering/codegen treats unions as byte-array aggregates where appropriate: static/local zero init, first-member union initialization plus padding, aggregate assignment/copy via CopyBytes, member access at offset 0, nested aggregate members, and aggregate conditionals.
- Also fixed a Chapter 18 validation gap found during review: by-value function definitions require complete parameter types, and function-call arguments must be complete by-value objects. Incomplete parameter declarations remain allowed and pass once the tag is completed before definition/call.
